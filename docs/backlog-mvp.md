# Backlog MVP - Sistema Taller Mecánico

## Reglas generales
- Cada tarea debe ser independiente
- No mezclar múltiples módulos
- No refactorizar fuera del alcance
- Mantener la app funcional en todo momento

---

# Sprint 1 - Base del sistema

## Tarea 1.1 - Layout y navegación
Crear estructura base de la app.

- layout principal
- sidebar
- header
- rutas principales
- páginas placeholder

---

## Tarea 1.2 - Tipos y mocks
Definir estructura de datos del sistema.

- types principales
- mocks realistas
- relaciones entre entidades
